Patrones estructurales

Los patrones estructurales son soluciones reutilizables para problemas comunes de diseño de software que se centran en cómo las clases y objetos se relacionan y componen para formar estructuras más grandes y complejas en una aplicación. Estos patrones ayudan a mejorar la organización y la flexibilidad del código. Ejemplos incluyen el patrón Adapter, Composite y Bridge, entre otros.

Adapter

El patrón estructural Adapter permite que dos interfaces incompatibles trabajen juntas al proporcionar una interfaz intermedia que traduce las solicitudes de una interfaz a otra.

Bridge

El patrón estructural Bridge separa una abstracción de su implementación, permitiendo que ambas evolucionen independientemente. Esto mejora la flexibilidad y la extensibilidad del diseño de software.

Composite

El patrón estructural Composite permite componer objetos en una estructura de árbol para representar jerarquías de objetos de manera uniforme, tratando objetos individuales y composiciones de objetos de la misma manera.

Decorator

El patrón estructural Decorator actúa como una envoltura flexible alrededor de un objeto, permitiendo agregar comportamiento o características adicionales sin modificar su código fuente.

Facade

El patrón estructural Facade proporciona una interfaz simplificada y unificada para un conjunto de subsistemas o clases, facilitando el uso de estos sistemas de manera más sencilla y ordenada.

Flyweight

El patrón estructural Flyweight se utiliza para minimizar el uso de memoria o recursos compartiendo eficientemente objetos similares y reduciendo la duplicación, lo que es útil en situaciones donde se manejan grandes cantidades de objetos similares.

Proxy

El patrón estructural Proxy actúa como un intermediario para controlar el acceso a un objeto o recurso, permitiendo agregar funcionalidades como lazy loading, control de acceso o monitoreo sin modificar el objeto original.

Patrones Creacionales

Los patrones creacionales son un conjunto de patrones de diseño en programación que se utilizan para la creación de objetos de manera flexible y eficiente, abstrayendo los detalles de cómo se instancian y se relacionan las clases. Estos patrones se centran en la creación de objetos, lo que ayuda a gestionar la complejidad de la creación de objetos en una aplicación.

Factory Method

El patrón creacional Factory Method define una interfaz para crear objetos, pero permite que las subclases decidan qué clase concreta instanciar. Esto proporciona flexibilidad para crear objetos de diferentes tipos dentro de una misma jerarquía de clases.

Abstract Factory

El patrón creacional Abstract Factory proporciona una interfaz para crear familias de objetos relacionados sin especificar sus clases concretas, permitiendo que un cliente cree objetos de manera coherente y flexible sin preocuparse por los detalles de implementación.

Prototype

El patrón creacional Prototype permite crear nuevos objetos duplicando un prototipo existente, lo que facilita la creación de objetos complejos y costosos, minimizando la duplicación de código y la carga en el rendimiento.

Singleton

El patrón creacional Singleton asegura que una clase tenga una única instancia y proporciona un punto de acceso global a esa instancia, lo que garantiza que haya una única instancia de la clase en todo el programa.

Patrones De Comportamiento

¿Que Es Patron De Comportamiento?

Los patrones de comportamiento son un conjunto de patrones de diseño en programación que se utilizan para gestionar la comunicación y el comportamiento entre objetos y clases. Estos patrones se centran en cómo los objetos interactúan y colaboran entre sí para lograr un comportamiento deseado en una aplicación.

Command

El patrón de comportamiento Command encapsula una solicitud como un objeto, permitiendo que los clientes parametricen las solicitudes, las encole o registren y admitan operaciones reversibles, lo que brinda flexibilidad y control en la ejecución de comandos.

Chain of Responsability

El patrón Chain of Responsibility permite pasar una solicitud a través de una cadena de objetos para que cada uno decida si la maneja o la pasa al siguiente.

Mediator

El patrón de comportamiento Mediator centraliza la comunicación entre objetos, evitando que interactúen directamente. En lugar de comunicarse entre sí, los objetos se comunican a través del mediador, lo que reduce las dependencias y mejora la modularidad.

Iterator

El patrón de comportamiento Iterator proporciona una forma de acceder a elementos de una colección secuencial sin exponer su estructura interna, permitiendo recorrer la colección de manera uniforme y eficiente.

Memento

El patrón de comportamiento Memento permite capturar y restaurar el estado interno de un objeto sin revelar detalles de su implementación, lo que facilita la reversión o restauración de un objeto a un estado anterior.

Observer

El patrón de comportamiento Observer permite que un objeto (el sujeto) notifique automáticamente a otros objetos (los observadores) cuando cambia su estado, lo que permite una relación de dependencia flexible entre ellos.

State

El patrón de comportamiento State permite que un objeto cambie su comportamiento cuando su estado interno cambia, creando una máquina de estados que gestiona las transiciones y acciones asociadas a cada estado.

Strategy

El patrón de comportamiento Strategy permite definir un conjunto de algoritmos intercambiables y encapsulados, que pueden ser intercambiados de manera dinámica en un objeto. Esto permite que un objeto varíe su comportamiento sin cambiar su estructura.

Template Method

El patrón de comportamiento Template Method define la estructura de un algoritmo en una clase base, permitiendo que las subclases implementen ciertos pasos específicos de ese algoritmo, manteniendo el orden y la estructura general inalterados.

Visitor

El patrón de comportamiento Visitor permite agregar operaciones o comportamientos adicionales a objetos de una estructura de clases sin modificar esas clases, al permitir que se agreguen visitantes externos que interactúen con los objetos.
